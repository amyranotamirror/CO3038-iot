## Commit convention
- `feat` – New feature
- `fix` – Bug fix
- `docs` – Documentation changes
- `style` – Code formatting (no logic changes)
- `refactor` – Code restructuring (without changing behavior)
- `test` – Adding or fixing tests
- `chore` – Maintenance tasks (e.g., updating dependencies)
